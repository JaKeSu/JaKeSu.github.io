1.看windows和linux能否相互ping通

Linux下用ifconfig查看ip是否正常
不正常的话
vi /etc/sysconfig/network-scripts/ifcfg-ens33（可用TAB自动补全）
查看网卡配置文件文件中的onboot=yes开机自启动 no 开机不启动

2.修改hostname主机名称

vi /etc/sysconfig/network
编辑HOSTNAME=Master
vi /etc/hostname
删除原来的内容，写入Master

3.查看防火墙状态

firewall-cmd --state查看防火墙状态
systemctl stop firewalld.service 停止当前防火墙
systemctl disable firewalld.service永久禁用

4.克隆slave

克隆两台虚拟机，分别命令为Slave0和Slave1

5.机群是否互通

分别相互ping三台机器，看是否相通
配置hosts文件，方便互相访问机群
vi /etc/hosts
文件末尾追加 master slave0 slave1之间的地址映射

scp /etc/hosts root@salve1:/etc/hosts远程传输到其他几台

6.配置无密码登陆

ssh-keygen 生成ssh公钥私钥串
cd ~/.ssh 进入生成的.ssh目录
ssh root@salve1 利用ssh远程登陆salve1 测试成功
scp id_rsa.pub root@slave0:~/.ssh/master.pub 利用远程传输将master的公钥传到slave0的.ssh目录下，并且改名为master.pub
cat master.pub >>authorized_keys 进入slave0，将master.pub的文件追加到authorized_keys（原来没有）下面
chmod 600 authorized_keys 更改authorized_keys的权限为600，600十进制为110 000 000 对应文件权限可读可写不可执行
11 查看详细信息

以上所有命令可以用ssh-copy-id root@slave1代替
最后完成master对master，slave0，slave1的无密码登陆，slave0对master,slave1对master的无密登陆

7.配置jdk和Hadoop环境

利用ssh secure shell client将jdk和Hadoop压缩包传输到master
进入压缩包目录，利用tar -zxvf 文件名解压文件到当前目录
vi ~/.bashrc编辑环境变量，配置环境
在~/.bashrc写入

export JAVA_HOME=/opt/jdk1.8.0_101
export PATH=$PATH:$JAVA_HOME/bin
export HADOOP_HOME=/opt/hadoop-2.6.4
export PATH=$PATH:$HADOOP_HOME/bin

保存并且退出
source ~/.bashrc编译
查看java与hadoop版本看是否成功

java -v
hadoop v
